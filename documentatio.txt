1. DATA STRUCTURES USED
-----------------------
* ktp_header: Appended to every UDP packet. Contains 'type' (Data/ACK), 'seq_num' (8-bit wrapping sequence), 'window_size' (piggybacked rwnd), and 'payload_len' (actual bytes of data).
* sender_wnd (swnd): Tracks the unacknowledged sequence base, the next sequence to send, and the receiver's window size limit.
* reciever_wnd (rwnd): Tracks the next expected in-order sequence number and the current receiver window size.
* Shared_Mem: The core IPC structure. It is an array of 100 elements, each representing a KTP socket. It holds the circular buffers (send_buffer/recv_buffer), process IDs, binding IP/Ports, lengths, and the swnd/rwnd states. Mutual exclusion is guaranteed via pthread_mutex_t.

2. FUNCTIONS
-----------------------
* ksocket.c:
  - k_socket(): Finds a free Shared_Mem block, initializes variables, and returns the index as an FD.
  - k_bind(): Stores the local port and destination IP/Port in the shared memory block.
  - k_sendto(): Copies user data into the shared memory send_buffer and increments send_count.
  - k_recvfrom(): Copies data from the in-order recv_buffer to the user application and frees the slot.
  - k_close(): Blocks until all pending data is sent, then gracefully shuts down the socket and clears memory.
  - dropMessage(): Simulates network loss based on the DROP_PROB threshold.

* initksocket.c:
  - R_handler(): The Receive Thread. Creates underlying UDP sockets for new entries. Uses select() to listen for incoming UDP traffic. Handles in-order data, buffers out-of-order data, drops duplicates, slides windows, and sends ACKs/duplicate ACKs.
  - S_handler(): The Send Thread. Wakes up every T/2 seconds. Handles timeouts (by rewinding next_seq_to_send) and loops through the send_buffer to transmit pending messages over UDP.
  - GC_handler(): The Garbage Collector. Uses kill(pid, 0) to check if the user process crashed. If it did, it frees the shared memory slot to prevent leaks.
